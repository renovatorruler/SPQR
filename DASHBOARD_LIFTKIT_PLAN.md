# LiftKit UI Integration Plan

## Scope
Integrate LiftKit components into the dashboard for:
- Live bot status (existing)
- Trade history (existing)
- Backtest results (new)

## New UI Sections

### 1) Backtest Summary (Dashboard)
- **LiftKit.Card** grid with key metrics:
  - Total return
  - Max drawdown
  - Win rate
  - Total trades
- Data source: `Backtest.result.metrics`

### 2) Equity Curve Panel
- **LiftKit.Card** wrapping a chart canvas (charting lib TBD).
- Data source: `Backtest.result.equityCurve`

### 3) Backtest Trades Table
- Reuse `TradeHistory` table styling with LiftKit Text/Heading.
- Data source: `Backtest.result.trades`

### 4) Strategy Parameters Panel
- **LiftKit.Card** displaying current QFL config values:
  - crackThreshold, base filter, exit policy, reentry policy, regime gate
- For now read from typed config (no form inputs).

## Components to Add
- `BacktestSummary.res`
- `BacktestEquityCurve.res`
- `BacktestTrades.res`
- `StrategyConfigPanel.res`

## Data Wiring
- Introduce a typed `Backtest.result` store (shared types) for the dashboard.
- Load from SQLite or a serialized file generated by the backtest runner.

## LiftKit Usage
- Use `LiftKit.Grid` for metric cards.
- Use `LiftKit.Card` for all panels.
- Use `LiftKit.Heading` / `LiftKit.Text` for labels and values.
